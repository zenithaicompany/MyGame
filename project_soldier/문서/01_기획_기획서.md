# Project Soldier 기획 기획서

## 1) 기획 목표
- 모바일에서 `직접 조작 FPS`의 손맛을 빠르게 검증한다.
- 오버워치 솔저형 전투 리듬(이동 사격 + 스킬 사이클)을 단일 캐릭터로 구현한다.
- 리소스 투자보다 조작 완성도와 전투 피드백에 집중한다.

## 2) 타겟 유저
- 모바일 FPS 조작에 익숙한 코어 유저
- 짧은 플레이 세션에서 즉각적인 타격감을 선호하는 유저

## 3) 핵심 재미 루프
1. 조이스틱으로 위치 선점
2. 우측 드래그로 에임 보정
3. 사격 버튼 홀드로 추적 사격
4. 상황에 따라 질주/로켓/생체장 선택
5. 전투 지속 → 생존/처치 피드백 획득

## 4) 전투 설계(초기안)
- 기본 사격
  - 히트스캔(Raycast), 연사형
  - 탄창/재장전 간소화 또는 임시 무한탄(초기 테스트 단계 선택)
- 질주
  - 진입/이탈이 빠른 포지셔닝 스킬
  - 공격 개시와 상충되도록 설계해 선택 비용 부여
- 나선 로켓
  - 중거리 견제/마무리 용도
  - 좁은 박스맵에서 범위 타격 체감 제공
- 생체장
  - 교전 지속력 확보용 생존 스킬
  - 위치 선정 판단을 요구하는 영역형 스킬

## 5) 콘텐츠 범위(MVP)
- 플레이어 캐릭터 1종
- 표적(Target) 박스 오브젝트
- 단일 테스트 맵(엄폐 박스 배치)
- 기본 HUD/버튼 UI

## 6) 제외 범위(후순위)
- 멀티플레이/매치메이킹
- 정식 캐릭터 모델링/애니메이션 고도화
- 고급 VFX/SFX 폴리싱
- 복잡한 AI 적 행동 트리

## 7) 밸런싱 기본 파라미터(초기 제안)
- 이동속도: `Base 5`, Sprint 시 `7.5`
- 사격: 초당 8~10발 범위에서 테스트
- 로켓: 재사용 대기시간 6~10초, 폭발 반경 3~5m
- 생체장: 5초 유지, 초당 20 회복

## 8) 플레이 테스트 질문
- 조준과 사격을 동시에 수행할 때 불편감이 없는가?
- 질주 해제 조건(사격 시 해제)이 직관적인가?
- 로켓/생체장 사용 타이밍이 자연스럽게 학습되는가?
- HUD 정보(탄창/HP)가 전투 중 즉시 인지되는가?

## 9) 수용 기준
- 10분 내 초보 테스터가 기본 조작을 학습 가능
- 전투 중 핵심 스킬 3종을 모두 1회 이상 자연스럽게 사용
- 조작 불편/입력 씹힘 관련 치명 피드백이 없을 것
