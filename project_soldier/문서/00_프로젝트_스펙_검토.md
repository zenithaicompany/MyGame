# Project Soldier MVP 스펙 검토 문서

## 1) 프로젝트 개요
- 프로젝트명: `Project Soldier`
- 엔진/렌더링: `Unity` + `URP`
- 개발 형태: 1인 개발
- 타겟 플랫폼: 모바일(Android/iOS)
- 장르/목표: 오버워치 `솔저:76` 감성의 하이퍼 FPS 조작감을 가진 경량 MVP

## 2) 핵심 컨셉 검토

### 컨셉 정리
- `솔저:76`의 이동/조준/사격 리듬을 모바일 UI 조작에 맞게 재구성
- 자동 사격 배제, 유저의 `직접 입력` 중심 전투
- 저비용 프로토타입을 위해 더미 캐릭터 + 박스형 레벨로 빠른 반복 개발
- 모바일 성능을 고려한 단순한 물리/이펙트 구조

### MVP 타당성
- 강점
  - 핵심 재미(에임 + 추적 사격 + 생존 스킬)가 빠르게 검증 가능
  - 리소스 부담이 낮아 1인 개발에 적합
  - 조작 체계를 먼저 확정해 후속 콘텐츠 확장에 유리
- 주의점
  - 모바일에서 `시점 조작`과 `사격 버튼 홀드`의 동시 입력 UX가 핵심 난점
  - 발열/프레임 저하를 막기 위한 효과 최적화 필요
  - 네트워크 기능 없이도 싱글 타격감 검증이 가능하도록 피드백 설계 필요

## 3) 기능 요구사항 검토

### A. FPS 핵심 컨트롤러
1. 이동
   - 좌하단 가상 조이스틱 입력 기반 이동
   - 디버그/PC 테스트용 `WASD` 병행 지원
2. 시점
   - 우측 터치 드래그로 카메라 회전(수평/수직 감도 분리 가능)
3. 수동 사격
   - 사격 버튼 `누르고 있는 동안` 연속 Raycast 발사
   - 사격 버튼 홀드 중 손가락 이동 시 에임 회전 지속 허용
4. 타격 반응
   - `Target` 태그 오브젝트 피격 시 일정 시간 빨간색으로 변환 후 복구

### B. 솔저형 스킬
1. 질주(Sprint)
   - 버튼 On 동안 이동속도 1.5배
   - 사격 시작 시 질주 즉시 해제
2. 나선 로켓
   - 버튼 클릭 시 전방으로 Sphere 발사(물리 기반)
   - 충돌 시 폭발 + `OverlapSphere` 범위 대미지
3. 생체장(Heal)
   - 플레이어 발밑에 원형 힐 존 생성
   - 존 내부 플레이어에게 5초간 초당 20 HP 회복

### C. UI/HUD
1. 조이스틱 에셋 연동
2. 사격/질주/로켓/생체장 버튼 바인딩
3. 중앙 크로스헤어, 하단 탄창/HP 텍스트 표시

## 4) 기술 구현 관점 권장사항
- 캐릭터 이동: `CharacterController` 기반 권장(모바일 MVP 안정성)
- 입력 체계: Unity `Input System` 또는 조이스틱 에셋 이벤트 + UI 버튼 이벤트 혼합
- 사격 주기 제어: 발사 간격(`fireRate`) 타이머로 관리
- 로켓/힐 존: 프리팹 기반 스폰 + 수명 타이머 자동 파괴
- 타격 반응: `Renderer.material` 직접 변경 대신 머티리얼 인스턴스 관리 주의(메모리 누수 방지)

## 5) 성능/품질 기준(초기)
- 목표 프레임: 중급 모바일 기준 60fps, 최소 30fps 방어
- Draw Call/실시간 라이트 최소화, 그림자 품질 낮춤
- 파티클/폭발 이펙트는 저비용 버전 우선
- 디버그 로그 과다 출력 제한

## 6) 리스크 및 대응
- 동시 입력 충돌(사격 + 에임): 터치 영역 분리 + 포인터 ID 기반 처리
- 조이스틱 오동작: 데드존, 최대 반경, 감도 튜닝 옵션화
- 스킬 밸런스 붕괴: 쿨다운/지속시간/회복량을 ScriptableObject로 분리해 조정 가능하게 설계

## 7) MVP 완료 기준(Definition of Done)
- 이동/시점/사격/피격 반응이 모바일 기기에서 끊김 없이 동작
- 질주/로켓/생체장 3종 스킬이 UI 버튼과 정상 연동
- HUD에 크로스헤어, 탄창, HP가 실시간 반영
- 5~10분 플레이 테스트에서 치명 버그 없이 반복 가능
