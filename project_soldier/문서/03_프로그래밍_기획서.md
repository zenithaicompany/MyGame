# Project Soldier 프로그래밍 기획서

## 1) 구현 목표
- 모바일 FPS 핵심 조작(이동/시점/수동 사격)을 안정적으로 구현
- 솔저형 스킬 3종(질주/로켓/생체장)을 독립 모듈로 구성
- UI 입력과 전투 로직을 느슨하게 연결해 유지보수성 확보

## 2) 시스템 아키텍처(초안)
- `PlayerController`
  - 이동 처리(조이스틱 + 키보드 폴백)
  - 카메라 회전(터치 드래그)
- `WeaponRaycastShooter`
  - 사격 버튼 홀드 감지
  - 발사 주기, Raycast 판정, 탄약 처리
- `PlayerSkillController`
  - Sprint 상태 관리
  - Rocket 발사
  - Heal Zone 생성
- `PlayerStats`
  - HP, 최대 HP, 회복/피해 처리
- `HUDController`
  - HP/탄약 텍스트 갱신
  - 크로스헤어 고정

## 3) 입력 처리 전략
- UI 버튼 이벤트
  - 사격: `OnPointerDown/OnPointerUp`로 발사 상태 전환
  - 질주: `OnPointerDown` 시작, `OnPointerUp` 종료(또는 토글형 선택)
  - 로켓/힐: `OnClick` 실행
- 시점 입력
  - 우측 터치 영역 전용 패널에서 드래그 델타 수집
  - 사격 버튼 홀드 중에도 별도 포인터가 시점 입력 가능하도록 분리

## 4) 핵심 기능 설계

### A. 이동/시점
- 이동 벡터 = 조이스틱 입력 + 키보드 입력(디버그용)
- 카메라 Pitch 제한(예: -70~70도)
- 프레임 독립성을 위해 `deltaTime` 기반 회전

### B. 수동 사격(Raycast)
- 발사 조건: `isFirePressed == true` && `time >= nextFireTime`
- 히트 시 `Target` 태그 검사
- 대상 오브젝트에 `IDamageable` 또는 `TargetHitReaction` 전달
- 질주 상태면 사격 시작 시 Sprint 자동 해제

### C. Target 반응
- 맞은 순간 재질 색을 빨강으로 변경
- 짧은 딜레이 후 원래 색 복원
- 연속 피격 시 코루틴 중복 처리 방지

### D. 나선 로켓
- 카메라 전방 방향으로 Sphere 프리팹 발사(`Rigidbody.velocity`)
- 충돌 시 폭발 로직 호출
- `OverlapSphere`로 범위 내 대상에 피해 적용 후 자폭

### E. 생체장
- 플레이어 발밑에 HealZone 프리팹 생성
- 트리거/거리 검사로 범위 내 플레이어 회복
- 5초 후 자동 제거

## 5) 데이터 파라미터 권장 분리
- 이동속도, 감도, 발사속도, 탄약, 스킬 쿨다운, 폭발 반경, 회복량 등을
  `ScriptableObject` 또는 직렬화 필드로 분리
- 코드 재컴파일 없이 인스펙터 튜닝 가능하도록 구성

## 6) 테스트 전략
- 단위 검증
  - 사격 홀드/해제 상태 전환
  - 질주 중 사격 시 강제 해제
  - 로켓 충돌 시 폭발/범위 대미지 발생
  - 힐 존 유지시간/회복량 검증
- 플레이 테스트
  - 양손 조작(이동+시점+사격 동시) 스트레스 테스트
  - 10분 연속 플레이 시 메모리/프레임 드랍 확인

## 7) 성능/안정성 체크 포인트
- `Update` 내 GetComponent 반복 호출 금지(캐싱)
- 물리 연산 최소화 및 레이어 마스크 적극 사용
- GC 유발 코드(문자열 연결/임시 할당) 억제
- 모바일 빌드에서 입력 지연/터치 누락 여부 확인

## 8) 완료 기준
- 핵심 컨트롤 + 스킬 3종 + HUD가 하나의 플레이 루프로 동작
- 기능 간 충돌(질주/사격/시점 입력) 없이 재현 가능
- 테스트 씬에서 반복 실행 시 치명 예외 로그 없음
