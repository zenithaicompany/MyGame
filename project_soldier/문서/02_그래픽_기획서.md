# Project Soldier 그래픽 기획서

## 1) 아트 방향성
- 목표: `가독성 우선`, `저비용 제작`, `고프레임 유지`
- 스타일: 단순한 형태, 선명한 색 대비, 기능 중심 시각 언어
- 기준: 오브젝트 역할이 첫눈에 구분될 것

## 2) 리소스 구성 원칙
- 캐릭터: 5등신 더미(실루엣 우선)
- 맵: 박스형 구조물 조합(엄폐/동선 테스트 목적)
- 재질: 단색 또는 저해상도 텍스처 최소 사용
- 조명: URP 기본 라이트 + 그림자 최소화

## 3) 오브젝트 시인성 가이드
- `Target` 박스: 기본색(중성) + 피격 시 빨간색 점멸
- 상호작용 가능 오브젝트는 채도/명도 대비를 높여 구분
- 배경/벽체는 저채도로 처리해 타겟 식별 방해 최소화

## 4) UI 시각 가이드
- 크로스헤어: 얇고 명확한 십자형(배경과 대비되는 색상)
- 버튼: 기능별 아이콘 단순화(사격/질주/로켓/힐)
- 정보 텍스트: HP/탄약은 큰 글자, 그림자 또는 아웃라인 적용
- 모바일에서 엄지 가림 영역을 고려한 배치

## 5) VFX 최소 사양
- 사격: 히트 포인트 간단 파티클 또는 디버그 임팩트
- 로켓: 발사 트레일 + 충돌 시 간단 폭발 이펙트
- 힐 존: 바닥 원형 데칼/반투명 원형 메쉬 + 은은한 펄스
- 성능 우선으로 파티클 수 제한

## 6) 제작 우선순위
1. 플레이 가독성 확보용 임시 머티리얼 세트
2. Target 피격 색상 반응
3. HUD(크로스헤어/HP/탄약) 시각 정리
4. 스킬 이펙트(로켓/힐) 최소 구현

## 7) 에셋 관리 규칙(권장)
- 폴더 분리: `Art/Characters`, `Art/Environment`, `Art/UI`, `Art/VFX`
- 네이밍: `PS_카테고리_용도_버전`
- 프리팹 기준축/스케일 통일(1유닛=1m)

## 8) 완료 기준
- 원거리/근거리에서 타겟과 배경이 명확히 구분
- 피격/폭발/힐 효과가 기능적으로 인지 가능
- 중급 모바일 기기에서 시각 효과로 인한 체감 프레임 저하가 크지 않음
